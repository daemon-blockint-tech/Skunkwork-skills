# Forward Deployed Engineer — Deployment Checklist (Senior/Chief Level)

Use this as a living checklist for every major deployment phase. Adapt to context.

## Phase 0: Pre-Engagement & Scoping
- [ ] Clear executive sponsor on customer side + mutual success criteria defined
- [ ] Access to relevant data sources, ontology (if exists), and operational users secured
- [ ] Initial ontology scan completed (core classes, relations, data quality snapshot)
- [ ] 1–3 candidate high-value workflows identified and prioritized with customer
- [ ] Risks (technical, organizational, change management) logged with mitigation owners

## Phase 1: Discovery (Week 0–1)
- [ ] Stakeholder map created (operators, data owners, IT, executives, ontology owners)
- [ ] Ontology & Value Map v1 produced and validated with customer
- [ ] Current-state pain points quantified where possible (time, errors, decisions)
- [ ] Quick-win scope locked (MVP ontology scope + first workflow)
- [ ] Technical environment access (dev/staging/prod-like) confirmed

## Phase 2: Rapid Prototyping (Week 1–3)
- [ ] Minimal viable ontology extensions designed and reviewed
- [ ] End-to-end prototype built with real or realistic data
- [ ] Provenance and audit logging implemented from day one
- [ ] Basic user interface / action surface tested with 2–3 power users
- [ ] Performance and data quality issues surfaced early
- [ ] Internal demo + customer demo completed with clear feedback captured

## Phase 3: Pilot Deployment & Adoption (Week 3–6)
- [ ] Pilot environment deployed (controlled production slice or pilot user cohort)
- [ ] Training materials + hands-on sessions delivered to pilot users
- [ ] Daily/weekly feedback rituals established (operators + ontology/product sync)
- [ ] Leading metrics instrumented and baseline captured
- [ ] Go/no-go criteria for broader rollout defined and agreed

## Phase 4: Value Expansion & Transition
- [ ] Ontology gaps and product feedback packaged and delivered to product team
- [ ] Customer team capability building plan executed (pairing, documentation, ownership transfer)
- [ ] Next value increment scoped with customer (new ontology module + workflow)
- [ ] Full deployment retrospective completed (what worked, what to improve, ontology health)
- [ ] Handover artifacts complete (runbooks, ontology documentation, decision log)

## Ongoing / Chief-Level
- [ ] Ontology health metrics tracked (coverage, consistency, usage, freshness)
- [ ] Product feedback loop cadence maintained and measured for actionability
- [ ] Customer self-sufficiency scorecard maintained
- [ ] Risks and technical debt actively managed and communicated

Print or copy this into your working notes for each engagement. Update it based on real deployment learnings.
